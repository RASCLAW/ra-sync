**What it is:** GitHub repo of pre-built design system markdown files mirroring popular site aesthetics (clean product-forward, editorial, SaaS-modern, etc.). Alternative entry point to Jack Roberts' Relume wireframe step when the target aesthetic is well-known.

**How to apply:**
- When duberymnl.com v2 build starts (Knockaround-inspired per backlog), pull the closest-matching design.md as starting spec for Claude Code to build against.
- Feed the chosen design.md into the v2 session CLAUDE.md as a reference artifact.
- No persistent context cost — one-time reference, not an installed skill.

**Locate via:** GitHub search for "awesome-claude-code-design" or similar (exact repo name referenced in Chase AI video, KjEFy5wjFQg at the awesome.md segment).

**Decision rule:** if target aesthetic is LOCKED (e.g., Knockaround clone) → awesome.md. If target aesthetic is undefined → Firecrawl + Relume (Jack Roberts workflow).
