**The frame:** Technical mapping (from Brock's video):
- Skills → "AI employees" (e.g., "your morning-briefing employee")
- Connectors → "data-source hookups" (Clarity, Gmail, Drive)
- Sub-agents → "specialist contractors"
- Projects → "shared workspaces for a specific job"
- Cowork itself → the Claude Desktop App's Cowork toggle

**Why it matters:** Solar installer clients (RAS Creative locked niche) don't want MCP server jargon. "AI employee that writes your morning briefing every Monday" lands. "A scheduled Claude Code routine with the Microsoft Clarity MCP connector" does not.

**Productization opportunity:** Cowork onboarding as a one-time setup service (~2 hrs/client) — CLAUDE.md + 1 morning-briefing skill + 3 connector stubs (Clarity, Gmail, GSC). Sells alongside the monthly retainer.

**How to apply:**
- Use "AI employee" framing in any RAS Creative sales copy, portfolio bullets, client-facing docs.
- Keep Cowork-flavored vocabulary out of technical docs (CLAUDE.md, memory) — this memory is for external communication only.
- When building client deliverables, structure them as Cowork-native (skills + connectors) so non-technical users can self-serve day-to-day.
