When generating a batch of content prompts, plan and write them SEQUENTIALLY -- not in parallel batches.

**Why:** Session 114 batch 001 wrote prompts in parallel groups of 3-4. RA noticed the outputs lacked variety because each prompt was written in isolation without seeing what the others picked. Sequential writing lets each prompt be informed by the previous ones -- different surface, different lighting, different location, different atmosphere.

**How to apply:**
- Before writing any prompts, lay out the full batch plan (randomizer output)
- Write prompt #1, note what banks were picked
- Write prompt #2, deliberately pick DIFFERENT bank options from #1
- Continue sequentially through the batch
- This is slower but produces visibly better variety in the final feed
- The self-check "variety bank combo differs from other images in batch" is easier to enforce sequentially
