**2026-05-20 update:** Strategy shifted from Messenger-first to dual-channel. Website orders were coming in organically without RA noticing -- v3 site is a real conversion channel (AOV ~P923 vs P598 Messenger). Both channels now active and tracked.

**Positioning link 2026-04-11 (session 104):** DuberyMNL's funnel is the live reference implementation for the RAS Creative SOLUTIONS offer. Everything learned here — ad → landing/DM → qualification → nurture → booking — ports directly into the service-business retainer product. See `project_positioning_locked.md`.

**Why Messenger-first was the original strategy:** 1 week of Traffic ads (P716 spend, 27.9K impressions, 507 LPV) produced zero sales. Filipino consumers in the P699 price range prefer chat-based ordering. But website orders proved viable at higher AOV.

**Current dual-channel setup:**
- Messenger: chatbot handles orders end-to-end, complex questions handoff to RA
- Website (duberymnl.com v3): full order flow, COD, nationwide
- Ads: no prices shown, CTA drives curiosity comments + DMs
- Auto-DM offers dual path: order via Messenger OR use code DUBERY50 on landing page
- Meta Pixel (ID: 1513349880261420) + Microsoft Clarity (ID: wts41ahyih) installed 2026-05-20
- Full funnel tracked: PageView → ViewContent → AddToCart → Purchase (all verified firing)

**How to apply:**
- Treat both channels as equal -- don't optimize for one at the expense of the other
- Pixel data will feed retargeting audiences and conversion optimization over time
- Clarity gives session recordings + heatmaps for UX improvements
