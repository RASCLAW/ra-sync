Some UGC scenarios are too narrow in their definition and produce visually identical outputs regardless of what the randomizer picks from variety banks.

**Why:** Session 112 (2026-04-12). CAFE_TABLE was generated twice with different randomizer picks (different lighting, different atmosphere). Both outputs looked the same: sunglasses on walnut table + iced coffee. RA noted "always same scene again and again." The scenario definition itself locks the visual -- "sunglasses on a cafe table, casual just sat down energy" only has one visual interpretation.

**How to apply:**
- Narrow scenarios (CAFE_TABLE, DASHBOARD_FLEX, DESK_SHOT) should be used sparingly -- max 1 per batch
- Consider expanding narrow scenario definitions to allow more visual variety (different surfaces, different prop combinations, different camera angles)
- The batch randomizer (backlog) should weight away from narrow scenarios when generating large batches
- Broad scenarios (SELFIE_OUTDOOR, OOTD_STREET, SUNSET_PRODUCT) handle randomized bank picks much better because the visual space is wider
